我和AJAX的纠结之旅

标签（空格分隔）： 未分类

  连续过了三天一头雾水的日子。
  AJAX这个小东西，让人云里雾里了好几天。今天就来说说它吧。顺便也记录下来给未来的我一些提示，免得再次遇到感觉似曾相识，却还是云里雾里不清不楚。

  Ajax 即“Asynchronous Javascript And XML”。异步 JavaScript 和 XML，是指一种创建交互式网页应用的网页开发技术。通俗来说，它是一种在无需重新加载整个网页的情况下，能够更新部分网页的技术，现在应用极为广泛。不刷新就能拿到新的数据？原来的网页那是想想都觉得不可能的，所有的数据更新都只能是在刷新之后，或者打开新页面跳转之后。

  这样的技术发展无疑给了用户更好的浏览体验。不过我个人觉得，这也使用户在不知不觉中将更多的时间花在了互联网上，日复一日，年复一年的在庞大新奇的新信息中翱翔，不知今夕是何夕。只要有能联网的设备，便有一种任你风云纵横，天下信息尽在我手，养成以上帝视角俯瞰他人的习惯。

  扯得有点远，继续回来说AJAX。

  ajax是由一系列老技术组成的不算新技术的技术，最早是由google将之发扬光大的，google将它用在搜索框上，实现了我们现在已经习以为常的联想关键词搜索。

  当然，任何一项技术都有利弊之说，ajax也不例外，大致列举了几项，如有问题，欢迎指出。

  ajax优点:
    1.减轻服务器的压力，按需获取数据,最大程度的减少冗余请求；
    2.局部刷新页面,减少用户心理和实际的等待时间,带来更好的用户体验；
    3.基于xml标准化,并被广泛支持,不需安装插件等；
    4.进一步促进了页面和数据的分离。

  ajax缺点:
    1.AJAX大量的使用了javascript和ajax引擎。
      这些取决于浏览器的支持，在编写的时候需要考虑对浏览器的兼容性；
    2.AJAX只是局部刷新,所以页面的后退按钮是没有用的。

  最后列举一个ajax的交互模型给大家，便于参考：

  ajax交互模型（打电话模式）
  1.有个电话
    new XMLHttpRequest
  2.给谁打（输入号码）
    ajax.open(‘get’,url,true);
  3.发送号码
    ajax.send()
  4.需要等待
    ajax.onload
  5.通话
    ajax.responseText

  好了，今天就写到这里了，已经过了午夜12点了。12点之前我没有变成公主，自然也没有水晶鞋和南瓜马车带我去参加华丽的舞会。所以我依然是个灰姑娘，只能乖乖的一直敲代码直到现在。
  自从开始认认真真的努力学习，就开始觉得一天二十四个小时根不能不够用嘛，明天见喽各位~~~
